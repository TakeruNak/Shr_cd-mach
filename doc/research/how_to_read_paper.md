# 論文の読み方

論文の読解はまだ試行錯誤の段階ですが，現時点で実践しているプロセスを順序立てて整理した.

## 読解中に少しだけ意識しておくこと
論文読解中に少しだけ意識しておくべきことを以下にリストする．

- **この論文で「何を学びたいのか？」**：
> 情報に対する目的意識を持つことは全体を通して意識しておくべき． この目的意識がないと，情報の取捨選択ができず，漠然として読んでしまう可能性がある．
- **できる限り全て読まないようにすること**：
> 論文は全てを読む必要はない．むしろ，必要な部分だけを選択して読むことが重要．

## 読解プロセス

### 1. タイトルとキーワードの確認
まずはタイトルを確認し，その論文が扱う主要なキーワードを把握．

### 2. Abstract（要約）の精読
Abstractを読み，以下の3点に重点を置いて内容を把握．

> * **Research Issue**: 解決すべき問題は何か？
> * **Statement / Hypothesis**: 問題に対し、どのような主張や仮説を立てているか？
> * **Methodology / Evidence**: その仮説をどのように検証しているか？

### 3. Summary / Conclusion（結び）の確認
文末のまとめを読み，改めて**Issue・Statement・Method**を確認．

### 4. セクション構成の把握
各章のタイトル（Heading）のみを流し読みし，著者がどこに力点を置いているか（構成の比重）を把握．

### 5. 図表（Figure / Table）の確認
図や表を先行してチェックし，主張を裏付けるデータや手法の具体的なイメージを掴む．

### 6. 本文の精読（強弱をつけた読解）
必要に応じて以下の順序・方法で読み進める．

* **重点箇所**: `Introduction`（背景）と `Discussion`（考察）を優先．
* **パラグラフ・リーディング**: 
    * 各段落の **Topic Sentence**（要旨）と **Supporting Sentence**（支持文）を意識．
    * Topic Sentence で内容が把握できれば，詳細は適宜読み飛ばすことも．

### 7. 抽象と具象の往復
理解を深め，自身の研究に繋げるために思考を整理．

* **具象 → 抽象**: 個別の主張を汎用的な概念に言い換える．拘束条件を外したり，変数を抽象化する．数式としてだけでなく，概念的なレベルでも．情報を引くイメージ．
* **抽象 → 具象**: 概念的な主張を具象的な事例に落とし込めないか考える．拘束条件を加えたり，変数代入などを行う．数式としてではなく概念としても．情報を足すイメージ．
* **メタ認知**: 「パラメタに対する自由度」を上下させる感覚で，他の論文との相関性や位置づけを考察．

「具象 → 抽象」のプロセスは，statementを短文で表現するにはどうすればいいかを考えることに対応．「抽象 → 具象」のプロセスは，statementの情報を継ぎ足して，より具象的な表現を入れることに対応．
